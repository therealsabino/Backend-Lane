# Lane - Event | Backend

## Overview

Lane is a application developed for a college project. It provides functionality for creating, managing, and searching events, serving as the core API for an event-based application.

The project focuses on building a structured and scalable backend using modern Java development practices with Spring Boot.

## Features

* Create new events
* Search and filter events
* Manage event data
* RESTful API design
* Structured backend architecture

## Technologies Used

* Java
* Spring Boot

## Architecture

The application follows a typical layered architecture:

* **Controller layer**: Handles HTTP requests and responses
* **Service layer**: Contains business logic
* **Repository layer**: Manages data persistence

## API Overview

Typical endpoints may include:

* `POST /events` – Create a new event
* `GET /events` – Retrieve all events
* `GET /events/{id}` – Get event by ID
* `GET /events/search` – Search events based on criteria
